# R6 Loadout DB

A community database of **Rainbow Six Siege** operators, weapons, and gadgets — with a dark-themed browser viewer and a Node.js build script that merges all data into a single, fully-resolved JSON file.

🌐 **Live site:** [https://bmat9000.github.io/r6-loadout-db/](https://bmat9000.github.io/r6-loadout-db/)

---

## 📁 Folder Structure

```
r6-loadout-db/
├── assets/
│   ├── operators/
│   │   ├── png/           ← operator icon PNGs
│   │   └── svg/           ← operator icon SVGs
│   ├── weapons/
│   │   ├── primary/       ← primary weapon images
│   │   └── secondary/     ← secondary weapon images
│   └── gadgets/
│       ├── generic/       ← generic gadget images
│       └── unique/        ← unique gadget images
│
├── data/
│   ├── operators.json     ← operator data (edit to add operators)
│   ├── weapons.json       ← weapon data (edit to add weapons)
│   └── gadgets.json       ← gadget data (edit to add gadgets)
│
├── output/
│   └── operators.full.json  ← generated by npm run build
│
├── scripts/
│   └── build-final-json.js  ← build script
│
├── index.html             ← GitHub Pages entry point (same as viewer.html)
├── viewer.html            ← browser-based operator viewer
├── style.css              ← dark theme styles
└── package.json
```

## 🚀 Getting Started

### View it live

Visit the live site at [https://bmat9000.github.io/r6-loadout-db/](https://bmat9000.github.io/r6-loadout-db/) — no setup required!

### For developers — use the data

1. **Clone the repo**
   ```bash
   git clone https://github.com/Bmat9000/r6-loadout-db.git
   cd r6-loadout-db
   ```

2. **Data files** are in `data/operators.json` and `data/weapons.json`

3. **Assets** are in the `assets/` folder

4. **Generate the merged JSON** (optional — `output/operators.full.json` is already committed):
   ```bash
   npm install
   npm run build
   ```

5. **Run locally**:
   ```bash
   npx serve .
   ```
   Then open [http://localhost:3000](http://localhost:3000) in your browser.

## ✏️ Adding Data

### Add a new operator

Edit `data/operators.json` and add an entry like:

```json
{
  "id": "thermite",
  "name": "Thermite",
  "side": "attack",
  "org": "FBI SWAT",
  "speed": 2,
  "armor": 2,
  "primaryWeapons": ["m1014", "556xi"],
  "secondaryWeapons": ["m45_meusoc", "5_7_usg"],
  "gadgets": ["stun_grenade", "claymore"],
  "uniqueGadget": "exothermic_charge",
  "icon": "assets/operators/png/thermite.png"
}
```

> Make sure all referenced weapon and gadget IDs exist in `weapons.json` and `gadgets.json`.

### Add a new weapon

Edit `data/weapons.json` and add an entry like:

```json
{
  "id": "556xi",
  "name": "556XI",
  "type": "Assault Rifle",
  "category": "primary",
  "firingModes": ["Full-Auto", "Single"],
  "damage": 47,
  "fireRate": 690,
  "mobility": 50,
  "capacity": 30,
  "image": "assets/weapons/primary/556xi.png"
}
```

### Add a new gadget

Edit `data/gadgets.json` and add an entry like:

```json
{
  "id": "exothermic_charge",
  "name": "Exothermic Charge",
  "type": "unique",
  "description": "Thermite's unique gadget. Destroys reinforced walls.",
  "image": "assets/gadgets/unique/exothermic_charge.png"
}
```

After adding data, run `npm run build` again to regenerate `output/operators.full.json`.

## 🖼️ Converting Weapon Images to PNG

To convert all `.avif` and `.webp` weapon images in `assets/weapons/` to `.png`, run:

```bash
npm install
npm run convert-images
```

The script will convert each file and delete the original, logging progress as it goes.
